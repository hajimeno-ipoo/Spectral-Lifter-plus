# ノイズ処理・補正・マスタリング棚卸しレポート

作成日: 2026-05-01

## 結論

いま直すべき最優先は、補正やマスタリング本体ではなく、`ノイズチェック` の測定定義です。

現在の補正とマスタリングの処理順は大きく破綻していません。ただし、ノイズチェックは多くの項目で「ノイズだけ」ではなく「その帯域にある音全体」を測っています。そのため、強い設定にしても表示値が動きにくく、マスタリング後に音量が上がるとノイズが戻ったようにも見えます。

今後は、特定音源に合わせた調整は行わず、測定定義、表示仕様、テスト基準を固定してから実装修正します。

## 現在の補正フロー

実装箇所:

- `Sources/VelouraLucent/Services/NativeAudioProcessor.swift`
- `Sources/VelouraLucent/Models/AudioProcessingModels.swift`
- `Sources/VelouraLucent/Views/CorrectionSettingsPanel.swift`

処理順:

1. 入力音声を読み込む
2. 音声を解析する
3. ノイズを除去する
4. 高域を補完する
5. ダイナミクスを整える
6. 最終音量を整える
7. ファイルを書き出す

現在の判断:

| 項目 | 判断 | 理由 |
|---|---|---|
| 処理順 | 維持 | 補正を前処理として使い、後段でマスタリングする流れとして自然 |
| ノイズ除去 | 要確認 | 実処理はあるが、現在の表示指標では効き具合を正しく判断しにくい |
| 高域補完 | 維持。ただし後で検証 | ノイズ除去後に高域を補う流れは成立するが、過補完の検出が必要 |
| ダイナミクス整理 | 要確認 | 強い設定でどの帯域がどの程度抑えられるかをテストで固定する必要あり |
| 最終音量補正 | 維持 | 補正後が入力より弱くなりすぎないようにする目的と合っている |

## 現在の補正設定

実装箇所:

- `DenoiseStrength.settings`
- `CorrectionSettings`

プリセット:

| プリセット | 目的 | 現在値の傾向 |
|---|---|---|
| 弱い | 音の芯を優先 | 原音保持が高く、ノイズ処理は控えめ |
| 標準 | 自然さと低減のバランス | 基準値 |
| 強い | ノイズをしっかり減らす | 補正強度、低域整理、検出感度が高い |

現在の判断:

- プリセット構造は残す。
- ただし、`強い` の実効差は、ノイズ種別ごとのテストで確認する。
- この音源だけに合わせて `強い` をさらに強くすることはしない。

## 現在のマスタリングフロー

実装箇所:

- `Sources/VelouraLucent/Services/MasteringProcessor.swift`
- `Sources/VelouraLucent/Models/MasteringModels.swift`
- `Sources/VelouraLucent/Views/MasteringSettingsPanel.swift`

処理順:

1. 音色を整える
2. ディエッサーで刺さりを抑える
3. マルチバンド圧縮で密度を整える
4. 倍音を足す
5. ステレオ幅を整える
6. ラウドネスとピークを整える

現在の判断:

| 項目 | 判断 | 理由 |
|---|---|---|
| 処理順 | 維持 | 一般的な仕上げ処理として大きな矛盾はない |
| 音色補正 | 要確認 | 高域を持ち上げるため、補正済みノイズが目立つ可能性がある |
| ディエッサー | 維持 | サ行・刺さり対策として必要 |
| 圧縮 | 維持 | 音量感を整えるため必要。ただしノイズ床も持ち上がる可能性がある |
| 倍音 | 要確認 | シュワシュワ感や人工感を増やす可能性がある |
| 音量 | 維持 | マスタリングとして必要。ただし比較時は音量差を補正して測る |

## 現在のノイズチェック

実装箇所:

- `Sources/VelouraLucent/Services/NoiseCheckReportService.swift`
- `Sources/VelouraLucent/Views/ContentView.swift`
- `Tests/VelouraLucentTests/NoiseCheckReportServiceTests.swift`

現在の表示項目:

| ノイズ種別 | 現在の主な測定 |
|---|---|
| ヒス・シュワシュワ | 8kHz-20kHz の平均スペクトル |
| サ行・シマー | 5kHz-14kHz の平均スペクトル |
| こもり・低いザラつき | 200Hz-1kHz の平均スペクトル |
| ハム・電源ノイズ | 50/60Hz と倍音付近の強い値 |
| 低域ゴロゴロ | 20Hz-150Hz の平均スペクトル |
| 環境音・部屋鳴り | 小音量区間の短期ラウドネス |

問題:

| 問題 | 確度 | 内容 |
|---|---|---|
| ノイズだけを測れていない | 高 | 平均スペクトルには楽器・声・余韻も含まれる |
| マスタリング後の音量上昇をそのまま受ける | 高 | ラウドネスが上がると、音楽成分もノイズ成分も値が上がる |
| 補正後も音量補正の影響を受ける | 高 | 補正も最後にラウドネスを戻している |
| 改善アドバイスが固定幅 | 高 | 測定原因に応じた助言ではなく、現在値に固定量を足している |
| テストが表示仕様寄り | 中 | 既知ノイズを検出し、処理後に下がることの保証が弱い |

## 正しい測定仕様

「ノイズらしさ」という曖昧な言葉は使わず、各項目に測定定義を持たせます。

| ノイズ種別 | 測定すべき値 | 比較時の注意 |
|---|---|---|
| ヒス・シュワシュワ | 小音量区間の 8kHz-20kHz ノイズ床 | 入力・補正後・マスタリング後を同じラウドネス基準にそろえる |
| サ行・シマー | 5kHz-14kHz の短時間ピーク突出量 | 平均ではなく、瞬間的な刺さりを見る |
| こもり・低いザラつき | 200Hz-1kHz の持続的な過剰成分 | 楽器の基音と混ざるため、全体音量との相対値で見る |
| ハム・電源ノイズ | 50/60Hz と倍音が周辺帯域よりどれだけ突出しているか | 絶対値ではなく周辺比を使う |
| 低域ゴロゴロ | 20Hz-150Hz の持続成分 | キックやベースの一時的な音とは分ける |
| 環境音・部屋鳴り | 小音量区間に残る広帯域ノイズ床 | 曲全体の RMS ではなく静かな区間を見る |

## 表示仕様

次の形に固定します。

| 表示列 | 意味 |
|---|---|
| 入力 | 元音源の測定値 |
| 補正後 | 補正後の測定値 |
| マスタリング後 | マスタリング後の測定値 |
| 補正での変化 | 入力から補正後でどれだけ減ったか |
| マスタリングでの変化 | 補正後からマスタリング後でどれだけ増減したか |
| 改善アドバイス | 問題がある時だけ、触る設定項目を出す |

表示値は2種類に分けます。

| 種類 | 用途 |
|---|---|
| 音量補正済みの比較値 | 補正やマスタリングで本当にノイズが減ったかを見る |
| 最終版の実測値 | 実際に聴いた時に目立つ可能性を見る |

## 実装計画

### 1. 現在の差分を固定する

目的:

- どこまで変更済みかを見失わないため。

作業:

- 補正設定
- マスタリング設定
- ノイズチェック
- 品質チェック
- UI
- テスト

完了条件:

- 変更済み・未確定・触らない箇所が文書化されている。

### 2. ノイズチェック用の解析を追加する

目的:

- `AudioMetricSnapshot` の平均スペクトルだけに頼らないため。

作業:

- 音声ファイル、または `AudioSignal` から直接ノイズ指標を計算する。
- 入力・補正後・マスタリング後を同じ測定式で解析する。
- 比較時は共通ラウドネスへ正規化する。

完了条件:

- 既知のヒス、サ行、ハム、低域ゴロゴロを個別に検出できる。
- ノイズなし音源を過剰に警告しない。

### 3. UIを測定定義に合わせて整理する

目的:

- 何を見ているか分かる表示にするため。

作業:

- `入力 / 補正後 / マスタリング後` を表示する。
- `補正での変化 / マスタリングでの変化` を表示する。
- 単位は項目ごとに明確にする。
- テキストは読みやすいサイズを維持する。

完了条件:

- 表示だけで「補正で減ったのか」「マスタリングで戻ったのか」が分かる。

### 4. 改善アドバイスを測定結果ベースにする

目的:

- 根拠の弱い固定幅アドバイスをやめるため。

作業:

- 補正後に問題がある場合は補正詳細を案内する。
- マスタリング後だけ問題が悪化した場合はマスタリング詳細を案内する。
- 両方に問題がある場合は、補正側を優先し、必要に応じてマスタリング側も出す。

完了条件:

- どの測定値が原因で、どの設定を触るべきかが説明できる。

### 5. 補正本体を触るか判断する

目的:

- 判定器が正しくなった後で、必要な場合だけ処理本体を変えるため。

作業:

- `弱い / 標準 / 強い` で、既知ノイズが段階的に減るかテストする。
- 減らない場合だけ `SpectralGateDenoiser` や `MultibandDynamicsProcessor` を調整する。

完了条件:

- 表示だけでなく、測定値として段階差が出る。
- ノイズなし音源を必要以上に削らない。

### 6. マスタリング本体を触るか判断する

目的:

- マスタリングで補正済みノイズを目立たせないため。

作業:

- マスタリング後にノイズ指標が悪化する場合だけ調整する。
- 高域、倍音、圧縮、音量のどの工程が原因かを見る。

完了条件:

- マスタリング後にノイズが不自然に戻らない。
- 仕上がりの音量・ピーク・聴きやすさは維持する。

## やらないこと

- 特定音源専用のしきい値調整。
- 表示だけ良く見せる修正。
- ノイズチェック修正前に補正本体をさらに強くすること。
- ノイズチェック修正前にマスタリングを弱くすること。
- 根拠のない固定幅アドバイスの追加。
