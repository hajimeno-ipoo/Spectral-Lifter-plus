# 解析診断ルート制による補正・マスタリング軽量化計画

## Summary
最初に素材を診断し、その診断結果から補正・マスタリングの各工程を `実行 / 軽量 / スキップ` に分けます。  
目的は、必要な処理は残し、不要な工程だけを安全に減らすことです。スキップ状態はログと進行状況に表示し、速度・音質・表示値の3つで完了判定します。

## Key Changes
- 新しく `ProcessingRoutePlan` を追加します。
  - 補正用: `CorrectionRoutePlan`
  - マスタリング用: `MasteringRoutePlan`
  - 各工程に `action: run / light / skip`、`reason`、`riskLevel` を持たせます。
  - ルートは処理開始直後に1回だけ決め、途中で判定が揺れないようにします。

- 補正ルートは `AnalysisData` と `NoiseMeasurementSnapshot` から決めます。
  - `lowNoiseCleanup`: `rumble` と `hum` が少ない時は `skip`。
  - `denoise`: 常に `run`。ノイズ除去本体は飛ばしません。
  - `sibilanceShimmerGuard`: サ行・シマーが少ない時は `light`、完全スキップはしない。
  - `harmonicRepair`: 常に `run`。高域補修は仕上がり差が出やすいため残します。
  - `repairShimmerGuard`: 高域補修前後のシマー危険が低い時だけ `skip`。
  - `lowMidResidueGuard`: 低中域ノイズが少ない時だけ `skip`。
  - `shimmerPeakLimit`: `hiss` と `shimmer` が少ない時は `skip`、注意域なら `light`。
  - `peakSafety`: 常に `run`。安全工程なので飛ばしません。

- マスタリングルートは `MasteringAnalysis` と補正後ノイズ測定から決めます。
  - `tone`: 常に `run`。
  - `deEss`: `harshnessScore` が低く、サ行ノイズも少ない時だけ `skip`。
  - `dynamics`: 常に `run`。音圧と密度に直結するため残します。
  - `saturate`: 設定値がほぼゼロなら `skip`。
  - `air`: 高域が十分あり、設定上の持ち上げも弱い時だけ `skip`。
  - `stereo`: 現在のステレオ幅と目標値が近い時だけ `skip`。
  - `loudness`: 常に `run`。
  - `highReturnGuard`: 高域戻りリスクが低い時は `skip`。
  - `noiseReturnGuard`: 常に入口は通すが、初回測定で問題なければ早期終了。

- 進行状況UIを4状態にします。
  - `未実行`: 通常のグレー丸
  - `実行中`: オレンジ
  - `完了`: 緑チェック
  - `スキップ`: グレーのマイナス
  - 進捗率は `完了 + スキップ + 実行中0.5` で計算します。

- ログ形式を固定します。
  - 補正: `ルート/補正: <工程名> = 実行|軽量|スキップ - <理由>`
  - マスタリング: `ルート/マスタリング: <工程名> = 実行|軽量|スキップ - <理由>`
  - 早期終了: `<工程名>: 早期終了 - <理由>`
  - 速度比較用に `ルート/補正/実行工程数: x/y`、`ルート/マスタリング/実行工程数: x/y` を出します。

## Completion Criteria
- 補正・マスタリングの全工程が、必ず `run / light / skip` のどれかに分類される。
- スキップされた工程はログと進行状況の両方に表示される。
- 軽い素材では、補正またはマスタリングで少なくとも1工程以上が安全にスキップされる。
- ノイズが多い素材、刺さりが強い素材、低域が濁った素材では、必要工程がスキップされない。
- `loudness`、`peakSafety`、`denoise`、`harmonicRepair` は常に実行される。
- 既存の品質チェック、ノイズチェック、比較グラフ、スペクトログラム表示が壊れない。
- 速度ログで、軽い素材では実行工程数または測定回数が減ったことを確認できる。

## Test Plan
- ルート判定テスト
  - クリーンな素材では、低域整理・シマー制限・ディエッサー・空気感などが条件に応じて `skip` になる。
  - ノイズが多い素材では、低域整理・シマー制限・ノイズ戻りガードが `run` になる。
  - 刺さりが強い素材では、サ行保護・ディエッサー・高域戻りガードが `run` になる。

- 進行状況テスト
  - スキップログを受け取ると `skippedSteps` / `skippedMasteringSteps` が更新される。
  - スキップ工程が進捗率に含まれる。
  - 入力変更・補正開始・マスタリング開始でスキップ状態がクリアされる。

- パイプライン回帰テスト
  - 軽い素材でスキップが発生しても出力ファイルが作られる。
  - 強いノイズ素材では必要工程がスキップされない。
  - `swift test` で既存の品質チェック・ノイズチェック・比較系テストがすべて通る。

- 速度確認
  - 既存の benchmark ログに `実行工程数` と `スキップ工程数` を追加。
  - 同じテスト素材で、軽い素材は実行工程数が減ることを確認。
  - 重い素材では工程を無理に減らさず、音質優先になることを確認。

## Assumptions
- 方針は安全優先です。速度のために音質リスクの高い工程は飛ばしません。
- UIに新しい設定スイッチは追加しません。軽量化は自動診断で行います。
- スキップ条件に迷う場合は `run` に倒します。
- STFT/ISTFTの大規模統合、ノイズ除去pass削減、工程順の入れ替えは今回の対象外です。
- 実装後の戻し方は、軽量ルート制のコミットを `git revert <commit>` で戻す形にします。
